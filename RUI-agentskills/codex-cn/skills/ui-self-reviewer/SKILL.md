# UI Self Reviewer

目标：在交付前做结构化自检，主动发现问题而不是等验收打回。

## SSOT

- `../../ui-self-evaluation.md`
- `../../AI_UI_Skill_Design_Document.md`
- `../../AI前端开发技能规范.md`
- `../contracts/quality-gates.md`

## 自审步骤

1. 代码自审
- 语法、类型、结构、命名、潜在死代码。

2. 视觉自审
- 层级、色彩、排版、间距是否一致。

3. 交互自审
- 状态完整性、反馈明确性、动效自然性。

4. 审美自审
- 现代感、细节完成度、品牌调性、差异化。

## 评分机制

- 每步输出分数与证据。
- 使用统一权重模型：
  - 完整性 30%
  - 美学 25%
  - 可维护性 25%
  - 性能 20%
- 输出分档结论：优秀/良好/合格/待改进/不合格。

## 重构触发规则

- 单文件 > 200 行
- 渲染逻辑 > 30 行
- 相同模式重复 >= 3
- Props 穿透 > 3 层

触发任一规则时，必须标记“需重构”并给出最短改造路径。

## 输出约定

1. 4 步自审结果
2. 总分与决策
3. 是否触发重构（是/否 + 原因）
4. 必修问题（阻断发布）
5. 增量优化项（可后续排期）

## 协作关系

- 在 `ui-acceptance-auditor` 之前执行，先做“内审”。
- 可与 `ui-aesthetic-coach` 联动，补审美与风格问题。
