# 需求澄清技能（精华版）

## 1. 目标
将模糊需求转化为可执行任务包，降低返工风险。

## 2. 输出物
- 需求摘要
- 关键问题清单
- 验收标准草案
- 风险与边界说明

## 3. 澄清顺序
1. 业务目标
2. 用户场景
3. 技术约束
4. 交付边界

## 4. 问题设计原则
- 一次只问高价值问题。
- 优先影响范围最大的未知项。

## 执行检查清单
- [ ] 输入前置条件已确认
- [ ] 关键约束已记录
- [ ] 输出格式可被下游技能消费
- [ ] 风险项与回滚思路明确

## 常见误区
- 只给结论，不给依据
- 输出不可执行或无法验证
- 忽略边界条件与异常路径
