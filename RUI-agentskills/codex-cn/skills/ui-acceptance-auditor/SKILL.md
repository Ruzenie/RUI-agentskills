# UI Acceptance Auditor

目标：把“看起来差不多”变成可量化验收。

## SSOT

- `../../ui-acceptance-criteria.md`
- `../../AI_UI_Skill_Design_Document.md`
- `../../AI前端开发技能规范.md`
- `../contracts/quality-gates.md`

## 三级验收

1. Level 0 基础合规（Pass/Fail）
- 语法、类型、响应式、基础交互、无阻断错误。
- 安全基线（输入转义、敏感信息暴露检查）与主流浏览器兼容性。

2. Level 1 质量标准（0-100）
- 视觉层次、色彩运用、排版质量、间距控制、组件质量。

3. Level 2 卓越体验（加分项）
- 微交互、设计亮点、响应式细节、可访问性增强。

## 执行流程

1. 先跑 Level 0
- 任一 fail 直接返回阻断问题，不进入后续评分。

2. 再跑 Level 1
- 输出维度分数 + 总分 + 最低分短板。

3. 最后评估 Level 2
- 作为增强项，不掩盖基础问题。

4. 输出五维验收矩阵
- 功能 / 视觉 / 代码 / 性能 / 安全与兼容。

## 判定建议

- `>= 85`：可发布（建议做小幅打磨）
- `70-84`：可用但建议修复短板后再发布
- `< 70`：建议继续迭代

## 输出约定

1. 验收结论（通过/有条件通过/不通过）
2. 分项评分表
3. 五维验收矩阵（功能/视觉/代码/性能/安全与兼容）
4. Top 5 问题与修复建议（按严重级）
5. 复验建议（只测哪些点）

## 协作关系

- 作为 `ui-codegen-master` 最后一道质量门。
- 与 `ui-self-reviewer` 联用形成“外部验收 + 内部自评”双保险。
