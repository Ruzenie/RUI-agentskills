# 组件示例库规范

## 概述

组件示例库是UI Agent Workspace的预制组件模板集合，提供标准化的组件画布模板，加速UI设计和开发流程。

## 目录结构

```
canvas/templates/
├── components/              # 组件模板
│   ├── button.md
│   ├── input.md
│   ├── card.md
│   ├── modal.md
│   ├── table.md
│   ├── dropdown.md
│   ├── tabs.md
│   └── ...
├── layouts/                 # 布局模板
│   ├── header.md
│   ├── sidebar.md
│   ├── grid.md
│   └── footer.md
└── pages/                   # 页面模板
    ├── login.md
    ├── dashboard.md
    ├── list.md
    └── detail.md
```

### Button（按钮）

```markdown
# Component: Button

## 元信息
- **类型**: component
- **分类**: 基础组件
- **复杂度**: simple
- **版本**: 1.0

## 组件概述
按钮是最基础的交互元素，用于触发操作或导航。支持多种样式变体、尺寸和状态。

**规格**:
| 属性 | 值 |

| 文字颜色 | #FFFFFF |

| 高度 | 40px |

┌─────────────────┐

**规格**:
| 属性 | 值 |

| 背景色 | transparent |
| 边框 | 1px solid #D1D5DB |
| 文字颜色 | #374151 |

---

### 变体3: Text（文字按钮）

**视觉图示**:

**使用场景**: 链接行为、低优先级操作、内联操作

**视觉图示**:

**使用场景**: 破坏性操作（删除、移除）

```

╰──────────────╯

## Props 接口

| 属性 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| variant | 'primary' \| 'secondary' \| 'text' \| 'danger' | 否 | 'primary' | 按钮变体 |
| size | 'sm' \| 'md' \| 'lg' | 否 | 'md' | 按钮尺寸 |
| disabled | boolean | 否 | false | 是否禁用 |
| loading | boolean | 否 | false | 是否加载中 |
| icon | ReactNode | 否 | - | 左侧图标 |
| onClick | (e) => void | 否 | - | 点击回调 |
| children | ReactNode | 是 | - | 按钮内容 |

## 使用示例

### 示例1: 基础用法
```tsx
<Button>提交</Button>
<Button variant="secondary">取消</Button>
<Button variant="danger">删除</Button>
```

### 示例2: 带图标
```tsx
<Button icon={<PlusIcon />}>新建</Button>
<Button icon={<SearchIcon />} variant="secondary">搜索</Button>
```

### 示例3: 加载状态
```tsx
<Button loading>保存中...</Button>
```

## 实现参考
- 代码: `src/components/Button/Button.tsx`
- 样式: `src/components/Button/Button.module.css`
```

## 使用模板

### 创建新组件画布

```markdown
基于模板创建:

1. 复制 `canvas/templates/components/button.md`
2. 重命名为 `canvas/components/{Component}.md`
3. 修改组件名称和内容
4. 更新元信息和规格
5. 绘制组件图示
```

### 模板变量

模板中使用以下变量占位符：

| 变量 | 说明 | 示例 |
|------|------|------|
| `{Component}` | 组件名 | Button, Input |
| `{component}` | 小写组件名 | button, input |
| `{category}` | 组件分类 | 基础组件, 表单 |
| `{author}` | 作者 | Agent |
| `{date}` | 日期 | 2026-02-16 |

## 最佳实践

1. **模板继承**: 基于现有模板修改，保持结构一致
2. **完整示例**: 每个组件至少3个使用示例
3. **状态覆盖**: 展示所有交互状态
4. **规格精确**: 尺寸、颜色使用具体数值
5. **响应式考虑**: 复杂组件说明响应式行为
6. **关联代码**: 始终链接到实现文件
