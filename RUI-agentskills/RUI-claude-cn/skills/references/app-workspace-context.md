# App 工作区上下文（精华版）

## 1. 用途
为 `app-workspace-guide` 提供本地参考快照，确保在真实 `app/` 缺失时仍可完成基线判断。

## 2. 建议读取顺序
1. 技术栈与构建信息
2. 目录约定与边界
3. 组件复用策略
4. 质量脚本与验收方式

## 3. 目录约束
- 页面层：负责路由和组合，不承载复杂业务细节。
- 组件层：可复用 UI 组件，具备稳定输入输出。
- 数据层：结构化配置和清单数据，避免散落硬编码。

## 4. 变更优先级
- 优先复用已有组件。
- 次选局部扩展。
- 最后新增抽象。

## 执行检查清单
- [ ] 输入前置条件已确认
- [ ] 关键约束已记录
- [ ] 输出格式可被下游技能消费
- [ ] 风险项与回滚思路明确

## 常见误区
- 只给结论，不给依据
- 输出不可执行或无法验证
- 忽略边界条件与异常路径
